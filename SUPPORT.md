# Support Policy

Thanks for using `conf`.

## Getting Help

- Usage questions and bug reports: open a GitHub issue in this repository.
- Include command, flags, platform, and sanitized output when possible.
- For sync correctness issues, include relevant diagnostics and whether the run was interactive or non-interactive.

## Response Expectations

- Best-effort maintainer support for community issues.
- No guaranteed SLA for response or resolution times.

## Supported Scope

- Current `main` branch behavior and the latest tagged release.
- Core commands: `init`, `pull`, `validate`, `diff`, `push`, `relink`.

## Not Covered

- Tenant-specific Confluence administration or policy setup.
- Recovery of deleted local files beyond Git history-based guidance.
- Custom forks with behavior diverging from upstream.
